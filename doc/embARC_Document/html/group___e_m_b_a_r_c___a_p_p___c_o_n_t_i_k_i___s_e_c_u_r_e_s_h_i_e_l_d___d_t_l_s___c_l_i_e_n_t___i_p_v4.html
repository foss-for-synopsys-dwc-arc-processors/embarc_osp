Detailed Description

embARC Contiki+Secureshield TinyDTLS dtls client ipv4 example

Extra Required Tools

Extra Required Peripherals

  • Digilent PMOD WIFI(MRF24WG0MA)

Design Concept

This example is designed to show how the secureshield is integrated with the contiki tinydtls dtls client. The pre-shared key, encryption and decryption related data (ccm.c, hmac.c and crypto.c) are allocated to container tinydtls. Background container can not access them without making a container call.

Usage Manual

This example requires to do some changes in dtls.c of tinydtls, where the dtls_retranmit_process needs to make a container call to do the retransmit. Please manually use git am <your-patch-file> to apply the 0001-necessary-xxxx.patch in current folder to tinydtls after run the apply_embARC_patch.sh.

  • You need to run git am <your-patch-file> in <embARC>/contiki/apps/tinydtls folder
    • command: git am ../../../../example/contiki/secureshield/dtls_client_ipv4/0001-necessary-changes-for-contiki-secureshield-tinydtls.patch
